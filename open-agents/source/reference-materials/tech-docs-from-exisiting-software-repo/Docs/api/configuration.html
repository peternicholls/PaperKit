Lightness Bias

Controls the overall brightness of your palette.

Option Effect Use When
.neutral Preserves anchor color lightness You want colors similar in brightness to your anchor
.lighter Shifts palette toward brighter colors (‚âà+0.2 in OKLab L) Light theme, airy design, high visibility
.darker Shifts palette toward darker colors (‚âà-0.2 in OKLab L) Dark theme, dramatic effect, low-light viewing
.custom(weight: Float) Fine control (range: 0.0-1.0) Precise lightness adjustment for brand guidelines

Temperature Bias

Shifts hue toward warm (red/orange/yellow) or cool (blue/green/purple) regions.

Bias Effect Use For
.cool Push hue toward blue/green/purple Professional, tech-forward, calming feel
.neutral No temperature bias Preserve natural hue progression
.warm Push hue toward red/orange/yellow Energetic, social, welcoming feel

Loop Mode

How the journey behaves at its boundaries (t=0 and t=1).

Mode Behavior Use For
.open Start and end are distinct colors Linear data, progress bars, sequential data
.closed Seamlessly wraps (t=1 matches t=0) Cyclic data, timelines, circular progress
.pingPong Reverses at boundaries (0‚Üí1‚Üí0) Symmetric patterns, back-and-forth animations

Mid-Journey Vibrancy

Boosts chroma (saturation) at the center (t=0.5) to prevent muddy midpoints.

Value Effect Recommended
0.0 No boost (linear saturation) When you want flat saturation progression
0.3-0.4 Subtle boost Most cases, gentle vibrancy
0.5-0.6 Moderate boost Default for most presets, vivid midpoint
0.7-1.0 Strong boost Special effects, energetic feel
‚ö° Pro Tip: If your 5-color palette looks dull in the middle, increase midJourneyVibrancy to 0.5-0.6.

Variation (Seeded Randomness)

Optional micro-variation that's deterministic (reproducible with same seed).

Option Effect Use When
.off No variation, fully deterministic You need exact, reproducible output
.subtle(dimensions, seed) Small variations, reproducible Organic feel with determinism
.noticeable(dimensions, seed) Larger variations, still deterministic Distinct variation, user-specific palettes

Variation Dimensions:

  • .hue ‚Äî Vary the color (hue angle)
  • .chroma ‚Äî Vary saturation
  • .lightness ‚Äî Vary brightness
  • .all ‚Äî Vary all dimensions together

Determinism with Variation:

  • Same seed = same variation (reproducible)
  • Different seed = different variation
  • No seed (random) = different every time

üîÑ Common Patterns

Pattern 1: Fixed Palette Generation

Generate a 10-color palette once, store it, use everywhere:

let masterPalette = ColorJourney( config: .singleAnchor(brandColor, style: .balanced) ).discrete(count: 10) // Store this and reuse throughout your app

Pattern 2: Dynamic UI Colors

Derive colors from user input on-the-fly:

func getColorsForTheme(_ theme: Theme) -> [ColorJourneyRGB] { let journey = ColorJourney( config: .singleAnchor(theme.baseColor, style: theme.style) ) return journey.discrete(count: 8) }

Pattern 3: Gradient Generation

Create smooth 256-step gradients for visualizations:

func generateGradient(from: ColorJourneyRGB, to: ColorJourneyRGB) -> [ColorJourneyRGB] { let journey = ColorJourney(config: ColorJourneyConfig( anchors: [from, to] )) return (0..<256).map { journey.sample(at: Float($0) / 255.0) } }
