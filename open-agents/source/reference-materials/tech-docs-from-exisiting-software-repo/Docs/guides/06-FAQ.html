üöÄ Getting Started

What is ColorJourney?

ColorJourney is a library that generates beautiful, perceptually uniform color palettes programmatically. It works by creating a "journey" through color space‚Äîstarting from one or more anchor colors and generating colors along a smooth, intentional path.

How is it different from other color libraries?

Most color libraries give you primitives (HSL, RGB conversions). ColorJourney is different:

  • Perceptually designed - Uses OKLab, a perceptually uniform color space
  • Deterministic - Same input always produces identical colors
  • Fast - Generates 10,000+ colors per second
  • Portable - Pure C core works anywhere (iOS, macOS, Linux, Windows, embedded)
  • Designer-friendly - Configure with intuitive terms like "warm", "vivid", "high contrast"

Do I need design experience to use it?

No! ColorJourney is designed for developers:

  • Presets - 6 built-in styles that work great out-of-the-box
  • Type-safe - Configuration enums prevent invalid combinations
  • Simple API - Just 3 basic operations: configure, sample, generate

What's the license?

ColorJourney is MIT licensed. You can use it freely in commercial projects.

üí° Usage & Configuration

What's the difference between sample() and discrete()?

Method Use Case Speed Output
sample(at: t) Single color at position ~0.6 Œºs One color
discrete(count: N) Distinct palette for UI ~0.1ms per 100 N distinct colors
linearGradient(stops: N) Smooth gradient in SwiftUI ~0.1ms per 100 Gradient object

What does "perceptually uniform" mean?

Colors that are mathematically equidistant appear visually equidistant to the human eye. In normal RGB space, a distance of (0.5, 0, 0) looks very different from (0, 0.5, 0). In OKLab (ColorJourney's internal space), equal distances always look equal to humans.

Can I use my own anchor colors?

Absolutely! Provide any RGB colors you like:

ColorJourneyRGB(red: 0.5, green: 0.2, blue: 0.8)  // Your color

What are loop modes for?

They control how the journey behaves at its boundaries:

  • .open - Start and end are different (good for linear gradients)
  • .closed - End loops back to start (good for circular color wheels)
  • .pingPong - Reverses at boundaries (good for bidirectional transitions)

What's the difference between lightness, chroma, and contrast?

  • Lightness - Overall brightness (shift darker or lighter)
  • Chroma - Saturation/colorfulness (muted or vivid)
  • Contrast - Minimum perceptual distance between colors (ensures distinctness)

Should I use variation?

Variation adds subtle randomness for organic feel:

  • No - Use .off for exact, reproducible colors
  • Yes - Use .subtle or .noticeable with a seed for organic but deterministic output

üî¨ Technical Details

What is OKLab?

OKLab is a perceptually uniform color space developed by Bj√∂rn Ottosson. ColorJourney uses it internally because distances in OKLab correlate with human perception of color differences.

Why not use LAB or LUV? OKLab is more recent and specifically designed for graphics/gamedev with better hue linearity and chroma predictions.

What is ŒîE (Delta E)?

ŒîE is perceptual distance between colors. In OKLab, it's just Euclidean distance: ‚àö(ŒîL¬≤ + Œîa¬≤ + Œîb¬≤). ColorJourney uses ŒîE to enforce minimum contrast between palette colors.

What's this "fast cube root" I see in the code?

OKLab conversion requires cube root calculations millions of times. The "fast cube root" uses bit manipulation + Newton-Raphson refinement to achieve ~1% error 3-5x faster than cbrtf(). The error is imperceptible to humans.

Is ColorJourney deterministic across platforms?

Yes, completely. Same config on iOS, macOS, Linux, Windows, or embedded systems produces identical colors. No floating-point variance issues.

‚ö° Performance & Optimization

How fast is ColorJourney?

  • Single sample: ~0.6 microseconds
  • Discrete palette of 100: ~0.1 milliseconds
  • 10,000+ colors per second
  • Zero allocations for continuous sampling

Should I worry about performance?

Almost never. Even generating 1000-color palettes takes <1ms. Only optimize if profiling shows ColorJourney is actually the bottleneck.

How do I optimize for better performance?

// 1. Create journey once, reuse it
let journey = ColorJourney(config: config)  // Create once
for i in 0..<1000 {
    let color = journey.sample(at: Float(i) / 1000)
}

// 2. Use sample() for gradients instead of discrete()
let gradient = journey.linearGradient(stops: 50)  // Optimized

// 3. Cache discrete palettes
let cachedColors = journey.discrete(count: 100)  // Cache this

What's the memory overhead?

  • Journey object: ~500 bytes
  • Discrete palette of N colors: N √ó 12 bytes (3 floats)
  • No hidden allocations or caches

Can I use ColorJourney in my game engine?

Yes! The C core can be used in:

  • Unity (via P/Invoke or native plugins)
  • Unreal Engine (via C++ binding)
  • Custom engines (via C/C++ FFI)
  • Any language with C interop

Does it work on older iOS/macOS versions?

The C core is fully portable. The Swift wrapper requires iOS 13+ / macOS 10.15+. For older versions, use the C API directly.

Can I use it on Android?

Yes, via NDK (native development kit). Link against the C core library.
